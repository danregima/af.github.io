# RegimA Africa Website Site Plan

## Overview
This document outlines the structure and organization of the RegimA Africa website (af.github.io).

## Site Structure

### Primary Pages
1. **Home** (`index.html`) - Main landing page
2. **About Us** (`about.html`) - Company information and background
3. **Products** (`products.html`) - Product catalog and services
4. **Blog** (`blog.html`) - News, updates, and articles
5. **Contact Us** (`contact.html`) - Contact information and forms
6. **FAQs** (`faq.html`) - Frequently asked questions
7. **Testimonials** (`testimonials.html`) - Customer testimonials and reviews

### File Naming Convention
- Use lowercase filenames with dashes for multiple words
- Use `.html` extension for all pages
- Main page should be `index.html`
- Keep names concise and descriptive

### Directory Structure
```
/
├── index.html (Home)
├── about.html (About Us)
├── products.html (Products)
├── blog.html (Blog)
├── contact.html (Contact Us)
├── faq.html (FAQs)
├── testimonials.html (Testimonials)
├── sources/ (Source files and assets)
├── assets/ (Images, CSS, JS)
└── docs/ (Documentation)
```

### Current Status
- Source files are located in the `Home/` directory
- Files need to be renamed according to naming convention
- Files need to be moved to appropriate locations

### Content Organization
- **Home**: Welcome message, overview, key features
- **About**: Company history, mission, vision, team
- **Products**: Product listings, descriptions, pricing
- **Blog**: Latest news, industry insights, company updates
- **Contact**: Contact forms, office locations, social media
- **FAQ**: Common questions and answers
- **Testimonials**: Customer reviews and success stories

### Technical Requirements
- Responsive design for mobile compatibility
- Clean URLs without spaces or special characters
- Proper meta tags and SEO optimization
- Fast loading times
- Accessibility compliance
